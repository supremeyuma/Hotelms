## ✅ Phase 1: Lock Down Access (VERY important)

Before adding features, make sure **staff can only see what they should**.

### 1️⃣ Decide how staff are identified

You likely already have:

* `users` table
* roles like `admin`, `staff`

**Confirm / enforce:**

* Admins → `/admin/*`
* Staff → `/staff/*`
* Staff **cannot** access admin routes

👉 If you don’t already have it:

* Use **Spatie Laravel Permission** or
* A simple `role` column on `users`

**Next task (recommended):**

* Create a `staff` middleware that:

  * Ensures user is authenticated
  * Ensures role = staff

---

## ✅ Phase 2: Department-Based Capabilities (Core Logic)

Staff belong to **one department** → now let departments **control features**.

### 2️⃣ Define what each department can do

Start simple:

| Department     | Can Do                                          |
| -------------- | ----------------------------------------------- |
| Administration | View reports, manage inventory, approve actions |
| Front Desk     | Check-in/out, bookings, room status             |
| Cleaning       | View assigned rooms, mark cleaned               |
| Security       | View shifts, log incidents                      |

👉 **Next task:**
Create a **DepartmentPermissions map** (code-based, not DB yet).

Example:

```php
return [
  'Administration' => ['inventory.manage', 'reports.view'],
  'Front Desk' => ['bookings.manage', 'rooms.view'],
  'Cleaning' => ['rooms.clean'],
  'Security' => ['incidents.log'],
];
```

This will later drive:

* What routes they can access
* What buttons they see in Vue
