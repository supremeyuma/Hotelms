# ✅ **Technical Project Description for AI Coding Assistant**

## **Project Name:** Holistic Hotel Management System (HMS)

## **Overview**

Build a full-stack hotel management system using **Laravel + Inertia.js + Vue 3 + Tailwind CSS**, fully compatible with **shared hosting on Hostinger**. The system should include a public booking website, an internal staff dashboard, per-room service ordering (via QR code links), role-based permissions, staff action-signoff using unique action codes, plus full operational modules (kitchen, laundry, housekeeping, maintenance, bookings, inventory, finance, reporting, and audit trails).

The system must run **without long-running processes or self-hosted websocket servers**, using only technologies compatible with shared hosting.

---

# 🏛 **Architecture Requirements**

### **Backend**

* **Laravel 10+**, PHP 8.1+ (Hostinger limit)
* MySQL / MariaDB
* Inertia server-side routing
* No separate API layer; use Inertia to pass props to Vue pages

### **Frontend**

* **Vue 3** (Composition API)
* **Inertia.js** (single-page navigation)
* **Tailwind CSS** for UI
* Optional small Alpine.js usage allowed but not required

### **Realtime Events**

* Use **Pusher** or **Ably** (hosted service)
* Do *not* use Laravel WebSockets or any solution requiring persistent processes

### **Queues & Jobs**

* Must use **database queue driver**
* No Horizon or long-running workers
* Use Hostinger cron job:

  * `php artisan schedule:run` every minute
  * `php artisan queue:work --once` every minute

### **File Storage**

* Default: local public disk
* Optional S3/DigitalOcean Spaces config allowed

### **Deployment Constraints**

* Must deploy fully on **Hostinger shared hosting**
* No Docker on production
* No Redis
* No broadcasting drivers needing sockets

# 🔐 **Authentication & Permissions**

### **User Account Types**

* Managing Director
* Manager
* Receptionist
* Kitchen Staff
* Laundry Staff
* Housekeeping Staff
* Maintenance Staff
* Other staff roles (permission-based)

### **Permissions System**

* Use **Spatie Laravel Permission**
* Roles control UI access, API access, and system operations

### **Staff Action Codes**

Each staff member has:

* A full **login account**
* AND a separate **action code** (4–8 characters)
* Stored hashed in DB (bcrypt/argon2)
* Used to “sign off” actions such as:

  * completing room cleaning
  * completing laundry
  * delivering room service
  * updating order status
  * closing maintenance ticket

The action-code authentication is different from login:

* It does **not** log the user in
* It only validates an action signature for audit purposes

# 🛏 **Modules Required**

## 1. **Public Website**

* Homepage
* Room listings
* Room detail pages
* Booking search (dates, guests, room type)
* Booking checkout + payment integration (Flutterwave/Paystack)
* Booking confirmation page
* Contact page
* Gallery page

## 2. **Booking Engine**

* Room availability checker
* Booking reservation with locking (transaction-safe)
* Payment redirection
* Webhook handler (booking status update)
* Email/SMS notifications
* Booking modification/cancellation

## 3. **Room-Specific Service Ordering**

Every room has:

* A unique URL
* A unique QR code pointing to that URL

Guest can order:

* Kitchen items (food)
* Laundry service
* Housekeeping requests
* Maintenance issues

Order lifecycle:
`Pending → Accepted → In Progress → Completed → Delivered`

Orders may be placed by:

* A guest on a device
* A receptionist on behalf of a guest
* Staff (internal)

## 4. **Staff Portal**

Staff can:

* Log in normally
* Access only the module(s) permitted by their role
* Perform departmental tasks
* Use quick-action mode:

  * select room/order
  * choose action (e.g., “Mark Cleaned”)
  * input **action code**
  * action is recorded in audit log

## 5. **Admin / Manager Dashboard**

Managers can:

* Manage bookings
* Manage rooms & room types
* Manage pricing
* Manage staff accounts and action codes
* Track active orders (kitchen, laundry, housekeeping, maintenance)
* Track inventory
* View audit logs
* View financial reports
* Manage site content (gallery, home page banners)

## 6. **Inventory Module**

* Track consumables (soap, towels, food stock, etc.)
* Alert for low quantities
* Staff consumption logs
* Room service consumption logs

## 7. **Laundry Module**

* Assign laundry orders
* Track statuses
* Mark items washed/delivered using action codes

## 8. **Kitchen Module**

* Menu items
* Kitchen orders queue
* Marking items prepared/delivered using action codes

## 9. **Housekeeping Module**

* Room cleaning queue
* Mark room as cleaned using action code

## 10. **Maintenance Module**

* Create maintenance tickets
* Assign staff
* Track ticket lifecycle
* Action code signoff on completion

## 11. **Finance Module**

* Payment logs
* Daily revenue
* Outstanding balances
* Booking receipts
* Export CSV/PDF

## 12. **Audit Trail System**

For each action:

* record actor (user_id or staff_id)
* record if action code was used
* record IP/device
* record timestamp
* record action type
* record before/after changes

Audit log must be immutable.
